# Advent of Code in Twinkle

A scaffolded workspace for solving [Advent of Code](https://adventofcode.com)
puzzles in Twinkle, test-first. The Makefile handles the busywork — fetching
inputs, generating test aggregators, running a day — so each puzzle is just a
`solution.tw` you grow against its sample.

Run every command from this directory (`examples/aoc`):

```bash
make help
```

## Layout

```
y2024/day01/
  solution.tw   parse / part1 / part2 / suite()   ← you edit this
  main.tw       reads input.txt, prints answers     (generated by `make new`)
  sample.txt    the puzzle's example                 (you paste it)
  input.txt     your personal input                  (pulled, gitignored)
  statement.md  the rendered problem text            (pulled, gitignored)
y2024/suite.tw  collects the year's day suites        (generated)
y2024/main.tw   runs the year                         (generated)
main.tw         runs every year                       (generated)
lib/aoc.tw      shared helpers (read_input, lines, ints, …)
template/       the skeleton `make new` copies from
```

Years are `y<YEAR>` and days `day<DD>` because a path segment must be a valid
module identifier (it can't start with a digit).

## Workflow

```bash
make new YEAR=2024 DAY=2     # scaffold y2024/day02 from the template
# paste the example block into y2024/day02/sample.txt,
# set the two `part_ sample` expectations in solution.tw to the example answers
make test YEAR=2024          # red — now implement part1/part2 until it passes

make pull YEAR=2024 DAY=2    # fetch your input + statement (needs a session cookie)
make solve YEAR=2024 DAY=2   # print the real answers
# record those answers in the `part_ input` expectations to lock in a regression
make test                    # green across every year
```

`make new` regenerates the aggregators, so a fresh day is picked up
automatically — Twinkle has no test auto-discovery, so `bin/gen.sh` writes the
`suite.tw` / `main.tw` files from whatever days exist on disk.

## Tests

Each `solution.tw` exports a `suite()` asserting `part1`/`part2` against the
committed sample and against the real input. The real-input checks are guarded
on `aoc.read_input` — they skip silently until you `make pull`, so a clean
checkout (with no inputs) stays green.

- `make test` — every year
- `make test YEAR=2024` — one year
- `make list` — every scaffolded day, grouped by year, flagging pulled inputs

`assert.tw` and `runner.tw` are symlinks to the canonical harness in
`boot/tests/`.

## Pulling inputs

`make pull` needs your AoC session cookie (the `session` value from your
browser cookies on adventofcode.com). Provide it either way:

```bash
export AOC_SESSION=<cookie>      # environment, or…
echo <cookie> > .session         # a gitignored file
```

Pulled inputs and statements are **gitignored**. Advent of Code asks that
puzzle inputs and text not be redistributed, so only your code and the small
public sample are committed.
